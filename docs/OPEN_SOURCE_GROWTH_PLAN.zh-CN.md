# 开源成长计划

高星项目通常不只是“有用”，还要做到容易理解、容易尝试、容易信任、容易贡献。RAW ISP RGB Lab 后续应该沿着这四个方向成长。

## 1. 让价值一眼可见

用户打开仓库 30 秒内应该知道：

- 这是什么：一个 RAW-to-RGB ISP 可视化学习实验室。
- 给谁用：图像工程初学者、ISP 学习者、作品集评审者。
- 为什么值得 star：它把公式、真实图像阶段、交互调参和性能洞察放在一个浏览器 demo 里。

## 2. 让 Demo 有记忆点

近期改进：

- 在 README 增加简短 demo GIF。
- 增加色卡、坏点、暗角、profiler 的截图展示。
- 在页面加入 “Try this first” 入门路径。
- 增加任务模式，例如修坏点、校暗角、恢复白平衡。

## 3. 让项目可信

当前基础：

- ISP 核心算法测试。
- 文档和 UI 结构 QA。
- GitHub Actions CI。
- 用户指南、技术设计、路线图、差距分析文档。

下一步：

- 增加 benchmark 报告。
- 增加图像质量指标。
- 增加更多算法参考资料。
- 增加真实 RAW 样例工作流。

## 4. 让贡献门槛低

已补齐的开源工程化内容：

- Issue 模板。
- Pull Request 模板。
- Roadmap。
- CI。
- 清晰仓库结构说明。

下一步：

- 增加 good first issue 列表。
- 增加“如何新增一个 ISP stage”的贡献示例。
- 增加代码风格说明。

## 6. 0.1 版本质量门槛

发布 0.1 前建议满足：

- README 中英文干净、没有乱码。
- Demo 有截图或 GIF。
- 所有内置案例都有测试覆盖。
- Profiler 在桌面和移动端布局正常。
- 用户指南诚实说明上传 RAW 的限制。
- Roadmap 现实、不夸大。
