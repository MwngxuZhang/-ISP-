# 教学助手升级：技术设计

## 概览

教学助手是一个确定性的前端功能，包含三部分：

1. 卡通讲解员图片资源：`assets/mascot-guide.png`
2. 结构化教学内容：`src/tutorial-content.js`
3. 教学面板和导航逻辑：`index.html`、`src/app.js`、`src/styles.css`

## 数据模型

每个教学步骤对应一个 ISP 阶段：

```js
{
  key: "demosaic",
  title: "Demosaic",
  plain: "把单通道 Bayer 马赛克变成 RGB 像素。",
  why: "相机传感器每个像素只采集一种颜色，所以缺失颜色需要估计。",
  watch: "颜色会出现，但边缘附近可能还有 zipper 或彩色伪影。",
  try: "点击 Demosaic 阶段卡片，然后故意切换 Bayer pattern，观察元数据错误的影响。"
}
```

这样可以让讲解内容可测试、可翻译，也便于后续扩展。

## UI 行为

- 教学面板显示卡通讲解员、当前阶段标题、白话解释、观察重点和推荐操作。
- 上一步 / 下一步按钮沿着 ISP pipeline 前进。
- 点击阶段卡片或 pipeline 轨道按钮时，教学面板同步更新。
- `Apply tip` 按钮可以应用少量安全的初学者预设。
- 当前选择的阶段也会显示在大预览中。

## 离线与开源约束

教学助手不调用外部 API。所有解释文字和行为都来自本地项目文件。

卡通讲解员是生成的 raster 资源，保存在仓库内。生成提示词会在最终说明中记录。

## 测试策略

自动化检查应该验证：

- 每个 ISP 阶段都有教学内容。
- 页面包含教学面板控件。
- 前端可以初始化教学助手。
- 现有 ISP 测试仍然通过。
- 文档链接仍然有效。
