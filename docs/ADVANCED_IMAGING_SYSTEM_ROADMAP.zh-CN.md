# 高级成像系统学习路线

本文档描述 PixelPipe ISP Lab 在现有 RAW -> ISP -> RGB 学习网站基础上的下一阶段技术路线。目标不是把项目做成简单的名词百科，而是把现代相机、手机、电影机、VR 眼镜、医疗成像设备和显示系统中的成像链路，拆成可以观察、可以调参、可以写代码、可以测试的学习模块。

## 1. 项目定位

PixelPipe ISP Lab 当前已经覆盖 RAW 数据、Bayer 排列、黑电平校正、坏点校正、镜头阴影校正、Demosaic、白平衡、颜色校正、去噪、锐化、Tone/Gamma、直方图、公式解释、源码 walkthrough 和性能分析。

下一阶段要从“单张 RAW 图像的 ISP 教学”扩展为“完整成像系统教学”：

- 成像控制：自动对焦、镜头控制、自动曝光、亮度和对比度控制。
- 颜色系统：自动白平衡、色温、色偏、颜色校正、显示适配。
- 设备场景：相机摄影、手机摄影、电影机、VR 眼镜、医疗成像、屏幕显示效果。
- 视频经验：视频编解码、时域去噪、视频恢复、增强、稳定性和实时性。
- 计算摄影与 AI：多帧融合、多目测量、投影几何、特征匹配、拼接、去模糊、去雾、超分、Transformer、GAN、扩散模型和 AIGC 辅助增强。

## 2. 总体学习地图

```text
Scene / Device
  -> Sensor and Lens
  -> Autofocus and Lens Control
  -> Auto Exposure and Tone Intent
  -> RAW ISP
  -> Color System
  -> Video / Multi-frame Processing
  -> Computational Photography / AI Enhancement
  -> Display and Evaluation
```

网站应把这条链路做成可点击的学习地图。每个模块都包含四类内容：

- 为什么需要：对应真实设备中的什么问题。
- 数学模型：核心公式、输入、输出和可调参数。
- 交互演示：拖动参数后，真实案例图像或视频片段如何变化。
- 代码实现：给出可读的 JavaScript 参考实现和进一步工程化方向。

## 3. 自动对焦系统

### 3.1 学习目标

自动对焦不是简单地“让图变清楚”，而是在镜头、传感器、场景和算法之间寻找一个最合适的焦平面。学习者需要理解：

- 镜头位置如何改变成像清晰度。
- 对比度自动对焦如何根据清晰度指标搜索峰值。
- 相位对焦、双像素对焦和深度辅助对焦为什么能加速搜索。
- 手机摄像头中的 VCM、OIS 和对焦马达如何与算法配合。
- 视频拍摄中为什么需要平滑对焦，避免呼吸效应和拉风箱。

### 3.2 核心模型

教学版可以先实现对比度 AF：

```text
focusScore = variance(Laplacian(luma))
bestFocus = argmax(focusScore(position))
```

镜头控制可抽象为：

```text
lensPosition(t + 1) = lensPosition(t) + step
```

其中 `lensPosition` 表示镜头位置，`focusScore` 表示当前画面的清晰度评分。搜索策略可以从粗到细：

1. 全范围粗扫，找到清晰度最高的大致位置。
2. 缩小范围细扫，得到更准确的焦点。
3. 视频模式下加入平滑滤波，避免焦点跳变。

### 3.3 网页交互设计

- 增加“对焦实验室”面板。
- 选择近景、远景、人像、低光、低纹理、运动物体等案例。
- 滑动“镜头位置”，真实图像跟随出现虚化或清晰变化。
- 显示清晰度曲线，标出当前点和最佳焦点。
- 提供“手动对焦”“自动粗扫”“自动细扫”“视频平滑对焦”模式。
- 吉祥物解释：为什么低纹理墙面难对焦，为什么夜景对焦慢，为什么手机会先拉一下再回到清晰位置。

### 3.4 测试指标

- 清晰度峰值能否被正确找到。
- 低纹理场景是否给出“不可靠”提示。
- 镜头位置变化是否平滑。
- 对焦失败案例是否能被教学解释，而不是只显示错误结果。

## 4. 自动曝光系统

### 4.1 学习目标

自动曝光决定画面亮不亮，也决定高光是否保留、暗部是否可见、噪声是否可接受。学习者需要理解：

- 曝光时间、模拟增益、数字增益、ISO 的区别。
- 亮度统计、直方图、ROI 测光和人脸优先测光。
- 亮度与对比度如何影响观感。
- 手机 HDR 为什么常常需要多帧曝光。
- 视频模式下曝光变化为什么要平滑，避免闪烁。

### 4.2 核心模型

基础 AE 可从目标亮度开始：

```text
error = targetLuma - measuredLuma
exposureGain = exposureGain * exp(k * error)
```

其中 `measuredLuma` 可以来自全图平均亮度，也可以来自中心区域、人脸区域或高光保护区域。

对比度控制可以使用简单曲线：

```text
out = (in - 0.5) * contrast + 0.5
```

高光保护可加入 clipping 约束：

```text
if highlightClippingRatio > threshold:
    reduceExposure()
```

### 4.3 网页交互设计

- 增加“曝光实验室”面板。
- 显示曝光三角：曝光时间、ISO/增益、光圈或等效光圈。
- 显示直方图、平均亮度、高光裁剪比例、暗部比例。
- 提供“全局测光”“中心测光”“人脸/主体测光”“高光优先”模式。
- 滑动场景亮度时，真实案例图像和直方图同步变化。
- 增加“电影感曝光”案例：保高光、压暗部、低对比 Log 风格。
- 增加“手机夜景”案例：用多帧合成说明为什么不能只拉高 ISO。

### 4.4 测试指标

- 目标亮度收敛是否稳定。
- 高光裁剪比例是否能被控制。
- 低光场景是否解释噪声代价。
- 视频曝光变化是否避免突然跳变。

## 5. 颜色系统

### 5.1 学习目标

颜色系统从白平衡开始，但不止白平衡。它还包括传感器色彩响应、色温估计、颜色校正矩阵、色域映射、肤色保护和显示端适配。

学习者需要理解：

- 色温和色偏的区别。
- Gray-world、White-patch、色卡校准和学习型 AWB 的差异。
- CCM 如何把相机 RGB 映射到目标色彩空间。
- 为什么手机会特别保护肤色、天空、草地等记忆色。
- SDR、HDR、P3、sRGB 和屏幕显示观感之间的关系。

### 5.2 核心模型

白平衡：

```text
R' = R * rGain
G' = G * gGain
B' = B * bGain
```

颜色校正矩阵：

```text
[R']   [m00 m01 m02] [R]
[G'] = [m10 m11 m12] [G]
[B']   [m20 m21 m22] [B]
```

色温到增益的教学映射可以先使用近似 LUT，让学习者理解趋势：低色温偏暖，高色温偏冷。

### 5.3 网页交互设计

- 增加“颜色实验室”面板。
- 提供钨丝灯、日光、阴天、荧光灯、混合光、人像肤色案例。
- 显示色温滑条、Tint 滑条、RGB gain 和色偏方向图。
- 显示色卡 before/after，标出灰卡误差和肤色偏移。
- 提供不同显示目标：sRGB、Display P3、HDR 风格预览。
- 吉祥物解释：为什么餐厅灯下会偏黄，为什么雪景容易偏蓝，为什么人脸不能只追求灰卡最准确。

## 7. 视频图像处理路线

### 7.1 视频编解码

学习内容：

- 帧内压缩和帧间压缩。
- I/P/B 帧、运动估计、码率、GOP、压缩伪影。
- 编码对细节、噪声和边缘的影响。

网页演示：

- 码率滑条：观察块效应、细节损失和边缘振铃。
- GOP 长度滑条：观察运动场景的稳定性。
- 编码前后差分图：显示压缩损失。

### 7.2 视频去噪

学习内容：

- 空域去噪与时域去噪的差异。
- 运动补偿为什么重要。
- 过强去噪为什么会造成拖影和塑料感。

网页演示：

- 单帧去噪 vs 多帧去噪。
- 运动物体拖影提示。
- 噪声强度、时域融合权重、运动阈值滑条。

### 7.3 视频恢复和增强

学习内容：

- 去模糊、去压缩伪影、超分、去雾、低光增强。
- 实时性、延迟、功耗和画质之间的取舍。

网页演示：

- before/after 滑杆。
- 指标面板：锐度、噪声估计、亮度分布、估算耗时。
- 教学提示：为什么算法看起来更清晰，但可能产生假纹理。

## 8. 计算摄影与 AI 图像路线

### 8.1 多帧融合和 HDR

- 多曝光融合：短曝光保高光，长曝光保暗部。
- 对齐：手抖和主体运动会造成鬼影。
- 融合权重：根据亮度、噪声和运动可靠性决定。

教学公式：

```text
out = sum(weight_i * alignedFrame_i) / sum(weight_i)
```

### 8.2 多目测量和投影几何

- 针孔相机模型、内参、外参、畸变。
- 极线约束、视差、深度估计。
- 手机多摄、VR 双目和机器人视觉中的应用。

教学公式：

```text
x' = K [R | t] X
depth = focalLength * baseline / disparity
```

### 8.3 匹配和拼接

- 特征点、描述子、匹配、RANSAC、单应矩阵。
- 全景拼接、视频稳定、多摄切换对齐。

教学公式：

```text
x2 ~ H x1
```

网页演示：

- 显示两张图的匹配线。
- 拖动错误匹配比例，看 RANSAC 如何剔除离群点。
- 显示拼接缝和融合区域。

### 8.4 去模糊、去雾和超分

- 去模糊：运动核、盲去卷积、学习型恢复。
- 去雾：大气散射模型、透射率、暗通道先验。
- 超分：从插值到 CNN、Transformer、扩散式增强。

去雾教学模型：

```text
I(x) = J(x) * t(x) + A * (1 - t(x))
```

其中 `I` 是雾图，`J` 是恢复图，`A` 是大气光，`t` 是透射率。

### 8.5 Transformer、GAN 和 AIGC

项目不应把 AI 模块做成黑盒按钮，而应强调输入、输出、损失函数和失败案例。

学习内容：

- CNN：局部卷积和纹理增强。
- Transformer：长距离依赖、窗口注意力、全局上下文。
- GAN：生成器、判别器、真实感和伪影。
- Diffusion / AIGC：逐步去噪、文本或条件引导、内容一致性风险。

网页演示：

- 展示“传统算法”和“AI 增强”的差异。
- 标出 AI 可能生成的假细节。
- 增加“可信度提示”：这是真实恢复，还是视觉上合理的生成。

## 13. 高级成像案例规格书

本阶段开发不只增加文档入口，而是把每个高级主题做成和现有 ISP 案例一致的学习单元：有图像输入、有可拖动参数、有 before/after、有公式、有代码思路、有失败提示、有设备解释。

### 13.1 每个案例必须包含的内容

| 字段 | 说明 | 网页表现 |

| 相机摄影 | 镜头焦平面、景深、色卡、RAW 宽容度 | 微距纹理对焦 + 色卡校准 | 镜头位置 / 微距步进 | 前后景清晰度峰值、灰卡是否中性 |
| 手机摄影 | 小传感器、逆光人像、HDR、肤色保护 | 逆光人像 HDR | 曝光 EV / 肤色暖度 | 人脸亮度、窗外高光、肤色是否自然 |
| 电影机 | 夜景灯牌、高光 roll-off、Log/LUT 风格 | 夜景街灯曝光 | 夜景曝光 EV | 灯牌是否过曝、暗部是否可读、对比是否稳定 |
| VR 眼镜 | 左右眼一致性、低延迟、畸变/色差校正 | 双目匹配与对齐 | 匹配置信度 | 左右结构是否错位、是否有重影 |
| 医疗成像 | 灰阶结构、信噪比、边缘稳定性 | 低光/灰阶结构增强 | 时域降噪强度 | 噪声是否降低、细结构是否被抹掉 |
| 屏幕显示 | sRGB/P3/HDR、Gamma/EOTF、环境光适配 | 色卡/逆光图显示映射 | 色温 / Tone | 灰阶、肤色、暗部和高光在显示端是否舒服 |

### 13.3 高级算法案例设计

| 技术点 | 输入图像 | 参数 | 正确变化 | 错误变化 |
| --- | --- | --- | --- | --- |
| 自动对焦 | 人像或微距纹理，前后景有明显深度 | 镜头位置 | 主体边缘变清晰，清晰度评分到峰值 | 低纹理找不到峰值，来回拉焦 |
| 自动曝光 | 逆光窗口、夜景街灯、高动态场景 | 曝光 EV | 暗部可见，高光仍有层次 | 窗外/灯牌裁成纯白，暗部噪声被放大 |
| 白平衡与颜色 | 色卡、混合光、人像肤色 | 色温 / Tint / RGB gain | 灰阶中性，肤色自然 | 灰卡准但肤色怪，或整体偏黄/偏蓝 |
| 视频降噪 | 低光连续帧或模拟噪声图 | 时域权重 | 随机噪声减少 | 运动主体拖影，纹理被抹平 |
| 编解码修复 | 低码率块效应图 | 去块强度 | 块边界减弱，边缘仍保留 | 去块过强导致塑料感 |
| HDR/匹配 | 多帧曝光或双目/全景重叠图 | 匹配置信度 | 对齐稳定，融合无明显鬼影 | 错配导致重影、拼缝、几何弯曲 |
| 超分辨率 | 1x 低分辨率模糊输入 | 2x/3x/4x 倍率 | 边缘和网格更细、更清楚 | 高倍率出现模型编造的假纹理 |
| 去雾/去模糊 | 雾天远景或运动模糊图 | 恢复强度 | 对比度、结构可读性提升 | 远处细节过度恢复、颜色不自然 |

### 13.4 网页布局要求

高级成像区域要和 ISP 区域分开：上半部分保持 RAW -> ISP -> RGB 学习链路，下半部分进入完整成像系统。高级区域的推荐顺序是：

1. 高级成像导语：说明本区学习自动对焦、曝光、颜色、视频、计算摄影和 AI。
2. 技术案例选择：AF、AE、颜色、视频、计算摄影、AI 等可切换实验室。
3. 当前案例主面板：左侧讲为什么、公式、代码；右侧显示 before/after、滑条、指标、观察点、失败点和完整链路说明。
4. 设备场景地图：用手机、相机、电影机、VR、医疗、显示六类卡片，把真实设备问题映射到具体案例。
5. 吉祥物说明：点击案例或拖动参数时，说明“这个参数对应真实设备中的哪个控制量”。例如曝光 EV 对应 AE 的目标亮度/增益控制，超分倍率对应 AI 重建尺度。

### 13.5 图像资产要求

- 所有内置图像必须能离线运行，存放在 `assets/demos/` 或未来的 `assets/advanced/`。
- 优先使用自制、生成或明确开源授权素材，避免直接使用不明来源照片。
- 如果使用合成图，也要尽量做成“场景化真实感”：逆光窗口、夜景灯牌、肤色、灰阶结构、左右眼视差、低分辨率纹理等。
- 每个图像资产要在文档中说明它模拟的真实设备问题，不能只作为装饰图。

### 13.6 验收标准

- 每个设备场景卡片必须有缩略图、设备说明、处理链路、可打开的实验室案例。
- 每个高级实验室必须有 before/after、滑条、指标、观察点、失败点和完整链路说明。
- AI 超分必须体现“1x 低清输入 -> 2x/3x/4x 更清晰输出”的关系，不能用普通亮度或锐化冒充。
- 医疗成像内容必须标注教学用途，不用于诊断。
- 页面和文档中不放项目包装、评分、差距分析类内容，只保留面向学习者的技术解释。

## 14. 项目边界

PixelPipe ISP Lab 是教学型开源项目，不声称替代生产级相机固件、医疗诊断系统或商业视频算法 SDK。生产级系统通常还需要硬件驱动、镜头标定、传感器标定、芯片 ISP、NPU/GPU 加速、功耗热稳定性、海量数据回归和严格质量评测。

本项目的价值在于把复杂成像链路拆成清晰的学习对象，让初学者能从“看见变化”走到“理解公式”，再走到“阅读代码”和“提出优化方案”。
