# 项目理解

## 目标

这个项目应该成为一个可以放到 GitHub 上开源的工具链，能够真实、可解释地跑通从传感器 RAW 数据，到 ISP 处理，再到 RGB 输出的完整流程。

目标用户是图像工程师、相机算法学习者和 ISP 初学者。他们需要看到每个阶段如何改变图像，而不只是阅读公式。

## 核心产品想法

构建一个用于 RAW 到 RGB 相机 pipeline 的可视化实验室：

- 用户可以加载类 RAW 的 Bayer 数据。
- 用户可以选择 Bayer pattern 和传感器元数据。
- 系统按阶段运行 ISP pipeline。
- 每个阶段都会显示图像预览。
- 关键参数可以在界面中调整。
- 同一套 ISP 函数可以被测试脚本或未来的命令行工具复用。

## MVP 中“完整流程”的含义

MVP 需要证明下面这条链路可以跑通：

```text
RAW Bayer samples
  -> black-level correction
  -> normalization
  -> demosaic
  -> white balance
  -> color correction
  -> gamma/tone mapping
  -> RGB image
```

这对学习已经足够完整，因为它包含了把单通道传感器采样转换成可显示彩色图像所需的最小 ISP 阶段集合。

## MVP 不声称解决的问题

MVP 不是完整的商业级 ISP。当前版本还不会针对以下内容做深度优化：

- 具体传感器的噪声模型
- 镜头阴影校准文件
- 自动曝光和自动白平衡测光
- 高质量边缘感知去马赛克
- 带 ICC profile 的完整色彩管理
- 与硬件 ISP 的一致性

这些能力会在教育版 pipeline 稳定之后逐步扩展。

## 开源定位

这个项目应该像一个干净、容易理解的学习型仓库：

- 易于 clone 和运行
- 不强依赖云服务
- 模块小，算法可读
- 对核心行为有测试
- 文档解释相机 pipeline 和工程路线

最适合的第一批用户，是那些希望先理解 ISP 概念，再进入更重的框架或生产级相机系统的人。
