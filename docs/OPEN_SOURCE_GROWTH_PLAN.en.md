# Open Source Growth Plan

High-star projects are usually not only useful. They are easy to understand, easy to try, easy to trust, and easy to contribute to. RAW ISP RGB Lab should grow along those four axes.

## 1. Make The Value Obvious

The first 30 seconds should answer:

- What is it? A visual RAW-to-RGB ISP learning lab.
- Who is it for? Image engineering beginners, ISP learners, and portfolio reviewers.
- Why star it? It connects formulas, real image stages, interactive tuning, and performance insight in one browser demo.

## 2. Make The Demo Memorable

Near-term improvements:

- Add a short demo GIF to README.
- Add a screenshot gallery for color chart, bad pixels, vignette, and profiler.
- Add a "Try this first" recipe on the page.
- Add task mode for common ISP learning goals.

## 3. Make The Project Trustworthy

Current foundation:

- Core tests for ISP algorithms.
- QA checks for docs and UI structure.
- GitHub Actions CI.
- User guide, technical design, roadmap, and gap analysis.

Next steps:

- Add benchmark reports.
- Add quality metrics.
- Add more algorithm references.
- Add real RAW sample workflow.

## 4. Make Contribution Low-Friction

Added project hygiene:

- Issue templates.
- Pull request template.
- Roadmap.
- CI.
- Clear repository map.

Next steps:

- Add "good first issue" list.
- Add CONTRIBUTING examples for adding a new ISP stage.
- Add coding style notes.

## 6. Quality Bar For A Stronger Release

Before calling the project 0.1:

- README has clean bilingual text.
- Demo has visible screenshots or GIFs.
- Built-in examples all pass tests.
- Profiler works in desktop and mobile layouts.
- User guide explains upload limits honestly.
- Roadmap is realistic and not overpromised.
